Camera parameters (internal orientation and distortion) after estimation:

Explanation:

Camera parameters provide the relationship between points in the 3D camera frame (object points) and the pixel coordinates in the image
In the idealized projection, where no distortion is present, the projection equation are as follows:
x' = x0 - c*(X'/Z')
y' = y0 - c*(Y'/Z')
where:
x' and y' are the measured image plane coordinates (the unit of which can be [px]) of the object point P(X',Y',Z') projected using the camera
with principal distance c and principal point (x0, y0). X', Y', Z' provide the position of point P in the 3D camera frame with the following definition:
For the horizontal, terrestrial image:
+X axis: towards left
+Y axis: upwards
+Z axis: backwards
In the presence of distortion the polynomial correction coefficients [k1, k2, k3, p1, p2] are estimated. This coefficients provide the
correction to the measured image coordinates, that should be distinguished from the case where the distortion coefficients provide the
displacement of the ideal coordinates to their measured locations.
The following observation equation is used for the estimation:
x' = x0 - c*(X'/Z') + x(k1*r^2 + k2*r^4 + k3*r^6) + p1*(r^2 + 2*x^2) + 2*p2*x*y
y' = y0 - c*(Y'/Z') + y(k1*r^2 + k2*r^4 + k3*r^6) + 2*p1*x*y + p2*(r^2 + 2*y^2)
where: x = x'-x0, y = y'-y0
and: r^2  = x^2 + y^2
This results in the following formulation of the correction to the measured image coordinates:
dx = - [x(k1*r^2 + k2*r^4 + k3*r^6) + p1*(r^2 + 2*x^2) + 2*p2*x*y]
dy = - [y(k1*r^2 + k2*r^4 + k3*r^6) + 2*p1*x*y + p2*(r^2 + 2*y^2)]
As a result:
x' + dx = x0 - c*(X'/Z')
y' + dy = y0 - c*(Y'/Z')

A number of photogrammetric and computer vision software handles the distortion in a different way:
once the object point is projected to the image to the P':[xp, yp] location (the 'ideal location'), the distortion parameters are used to
shifts the P' point to the place where it occurs in real  the image
This report provides the second set of distortion coefficients for each calibrated camera, that corresponds to the above
mentioned case. The applied mathematical model can be found in OpenCV documentation: https://docs.opencv.org/3.4.6/d9/d0c/group__calib3d.html
However this software does NOT use OpenCV to estimate for OpenCV distortion coefficients. The important feature of the OpenCV standard is the
orientation of camera frame axes, which follows:
For the horizontal, terrestrial image:
+X axis: towards left
+Y axis: downwards
+Z axis: looking forward, along the lens axis
An object point P (X, Y, Z) projects to the image according to the equation (notice the difference in sign):
x' = x0 + c*(X'/Z' + dx')
y' = y0 + c*(Y'/Z' + dy')
where: 
xp = c*(X'/Z' + dx')
yp = c*(Y'/Z' + dy')
represent the fiducial position of the undistorted point location (unit: [px]), x0 and y0 are the principal point coordinates (dimension: [px]),
dx' and dy' are the distortion corrections.
The values of dx' and dy' function of the estimated distortion coefficients and the normalized point coordintaes xn and yn:
xn = X'/Z'
yn = Y'/Z'
dx' = xn*(k1'*r^2 + k2'*r^4 + k3'*r^6) + 2*p1'*xn*yn + p2'*(r^2 + 2*xn^2)
dy' = yn*(k1'*r^2 + k2'*r^4 + k3'*r^6) + p1'*(r^2 + 2*yn^2) + 2*p2'*xn*yn
where r^2 = xn^2 + yn^2
k1', k2', k3' are  radial distortion parameters,
p1', p2' are tangential distortion parameters
we use the "'" upper index to distinguish from the distortion model described beforehand
As mentioned before this software does not use OpenCV to estimate for OpenCV distortion coefficients.
Here the surface fitting approach is applied: we can define a grid of points and calculate the distortion corrections dx', dy' for
each grid node. Having N grid nodes we can form 2*N linear equations for dx' and dy' and estimate distortion parameters
using the least squares method.

Estimated external orientation parameters of images:
Explanation:
The projection of object point to the image plane was defined by the following equation (see the camera calibration section):
x' + dx' = x0 - c*(X'/Z')
y' + dy' = y0 - c*(Y'/Z')
Here object point 3D coordinates: X', Y', Z' are provided in the camera 3D reference frame
External orientation parameters give the relation between camera reference frame and external reference frame
The relationship between object point coordinates in the camera frame and object point coordinates in the external frame
are given by the following equations:
X = r11*(X-X0) + r21*(Y-Y0) + r31*(Z-Z0)
Y = r12*(X-X0) + r23*(Y-Y0) + r32*(Z-Z0)
Z = r13*(X-X0) + r23*(Y-Y0) + r33*(Z-Z0)
where X0, Y0, Z0 are the coordinates of the image projection center
rij are the entries of the 3 X 3 rotation matrix R
The rotation matrix acts as follows: having the 3 element vertical vector v, coordinates of which are expressed in the
camera frame, the transformation V = R*v produces 3 element vector V that coordinates are expressed in the external reference frame
Rotation matrix can be seen as a composition of three 3-element vertical vectors i, j, k: R = [i j k], where each of i, j, k vector
is the unit vector parallel to the respective x, y, z axis of the camera frame that coordinates are provided in the external frame (direction cosine interpretation)
With the aim of estimation, the rotation matrix is parametrized in various ways.
The following parametrizations are used by this software:
1. Euler angles omega - phi - kappa:
    |   1       0             0     |   |  cos(phi)   0   sin(phi) |   | cos(kappa)   -sin(kappa)    0 |
R = |   0   cos(omega)  -sin(omega) | * |      0      1       0    | * | sin(kappa)    cos(kappa)    0 |
    |   0   sin(omega)   cos(omega) |   | -sin(phi)   0   cos(phi) |   |     0             0         1 |
2. Euler angles alpha - nu - kappa:
    | cos(alpha)   -sin(alpha)    0 |   |   1       0        0     |   | cos(kappa)   -sin(kappa)    0 |
R = | sin(alpha)    cos(alpha)    0 | * |   0   cos(nu)   -sin(nu) | * | sin(kappa)    cos(kappa)    0 |
    |     0             0         1 |   |   0   sin(nu)    cos(nu) |   |     0             0         1 |
3. Unit quaternion:
    | a^2+b^2-c^2-d^2   2*(b*c-a*d)      2*(b*d+a*c)   |
R = | 2*(b*c+a*d)    a^2-b^2+c^2-d^2     2*(c*d-a*b)   |
    | 2*(b*d+a*c)       2*(c*d-a*b)    a^2-b^2-c^2+d^2 |
where q = a + i*b + j*c + k*d
where i*b, j*c, k*d are the imaginary components
